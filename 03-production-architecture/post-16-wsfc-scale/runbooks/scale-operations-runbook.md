# Scale Operations Runbook

## Common Procedures

- Add node
- Remove node
- Expand storage presentation
- Patch with CAU
- Rebalance CSV ownership
- Review anti-affinity after workload changes

## Runbook Rule

At scale, every repetitive action needs a named owner, a rollback path, and a validation step. If you cannot explain how you reverse a large-cluster change, it is not ready for production.
